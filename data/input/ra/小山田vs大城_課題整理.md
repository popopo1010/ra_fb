## 1. 評価基準（マニュアル）

### 営業新規架電マニュアル

| Phase | 基準 | チェックポイント |
|-------|------|------------------|
| **Phase 1 受付突破** | 30秒以内に人事担当へ接続 | 開き方に「人数・年齢・資格」が具体的に入っているか |
| **Phase 2 接続後** | 相手のメリットを先に伝える | 「〇名いる」「強みがある」を先に。ヒアリングはそのあと |
| **Phase 3 ヒアリング** | 採用条件を網羅 | 職種・人数・年収・エリア・資格・経験・出張・残業・休日 |
| **Phase 4 料率・クロージング** | 35%ベース、履歴書送付の約束 | 料率を先に触れる。次のアクションを明確に |

### 受付突破の基本スクリプト（マニュアル）

```
「電気工事士で20〜30代の方、7〜8名ほどご紹介できる方がおりまして、
 いま人事ご担当の方いらっしゃいますでしょうか？」
```

**ポイント**: 具体的な数字（7〜8名）、年齢（20〜30代）、プル型の伝え方

### PSS（オープニング・プロービング・サポーティング・クロージング）

| 観点 | 基準 |
|------|------|
| **オープニング** | 用件を具体的に伝え、取次の価値を明確にする |
| **プロービング** | 相手のニーズを聞く。ヒアリングの順序・バランス |
| **サポーティング** | 相手のメリット（人材の強み・人数）を先に伝える |
| **クロージング** | 次のアクション（打ち合わせ・送付）を明確にする |

### 初回面談チェックリスト

1. 採用優先度（職種・人数・期限）
2. 対象エリア・出張
3. 働き方（休日・残業）
4. 資格・経験・年齢条件
5. 年収レンジ

## 3. 小山田の課題（既知）

### 開き方

1. 「人材紹介の件でご連絡したんですけれども」→ 抽象的
2. 人数・年齢・資格が入っていない
3. 受付に「社名は？」「何の用件ですか？」と確認が入りやすい

### 接続後

4. 「採用ニーズをお聞きしたい」が先 → 相手のメリットを伝える前にヒアリング
5. 相手が「紹介料はどれくらい？」と先に聞く＝相手主導

### 細部

6. メールアドレス誤り（ゼニタカ）
7. 担当者名の聞きそびれ

## 5. マニュアル基準の改善ポイント（共通）

| 観点 | マニュアル基準 | 改善の方向性 |
|------|----------------|--------------|
| 1. 開き方 | 7〜8名、20〜30代を具体的に | 抽象的表現を避ける |
| 2. 接続後 | 相手のメリットを先に | ヒアリングの前に「合いそうな方が〇名」 |
| 3. ヒアリング | 職種・人数・年収・エリア等を網羅 | チェックリストに沿って確認 |
| 4. クロージング | 次のアクションを明確に | 打ち合わせ・送付の約束 |
| 5. 細部 | メールアドレス復唱、担当者名聞き取り | 送付前確認、その場で聞き取り |
