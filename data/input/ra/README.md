# 初回架電（RA）文字起こし

5分以上話せた架電の音声・文字起こしを格納。FB 生成時に `ra_fb/feedback.py` が参照。

## ディレクトリ構成

```
ra/
├── README.md
├── 受付突破_断りパターンと繋ぎ方.md   # 受付突破時の断りパターンと繋ぎ方（FB参照）
├── 茂野vs小山田_課題整理.md           # RA別課題整理（FB参照・kadai）
├── 茂野/                              # 茂野・重野・大城（同一人物）の架電記録
│   ├── README.md
│   ├── 茂野_大城_課題整理.md
│   └── *.md
├── 小山田/                            # 小山田の架電記録
│   ├── README.md
│   ├── 小山田_課題整理.md
│   └── *.md
├── 小山田vs大城_課題整理.md           # 参考：課題整理
├── 茂野vs小山田_アウトプットイメージ（実サンプル）.md
├── 初回架電FB_サンプル（サンケイビル_小山田）.md
├── Slack_FB_アウトプットイメージ.md
└── Slack_FB_サンプルデータ.md
```

## RA別の格納

| フォルダ | RA | 説明 |
|----------|-----|------|
| **茂野/** | 茂野・重野・大城 | **同一人物**。名乗りにより表記が異なる（茂野／重野／シゲノ／大城） |
| **小山田/** | 小山田 | 小山田さんの架電音声・文字起こし |

## 受付突破の整理

初回架電において**受付突破がどれだけできるかが肝**。以下のドキュメントを参照：

- **[受付突破_断りパターンと繋ぎ方.md](受付突破_断りパターンと繋ぎ方.md)**
  - 断りパターン6種と繋ぎ方
  - 音声別・受付突破の整理（重野・茂野・小山田）
  - 音声を聞きながら参照することを想定

## 格納方法

### 音声ファイル

- **フォルダ**: 各RAフォルダ（茂野/、小山田/）に直接アップロード
- **形式**: .mp3, .m4a, .wav 等
- **ファイル名**: `{企業名略}_{連番}.mp3` 例: `TOKYO_KEIKI_001.mp3`

### 文字起こし・エッセンス

- 各架電ごとに1つのMarkdownファイル
- ファイル名: `{企業名略}_{連番}.md` 例: `TOKYO_KEIKI_001.md`
- 内容: メタデータ、文字起こし、抽出エッセンス、受付突破のポイント

## テンプレート

各RAフォルダの `_template.md` をコピーして使用（必要に応じて作成）。

## データの流れ

```
音声ファイル → 各RAフォルダに格納
       ↓
文字起こし（手動 or 音声認識）
       ↓
エッセンス抽出、受付突破のポイントを記録
       ↓
success_cases/ や マニュアル反映シート への活用
```
