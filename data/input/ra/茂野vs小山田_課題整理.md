# 茂野 vs 小山田：課題整理

> 茂野さんの方が受付突破率・接続後の獲得率が高い。小山田さんの両方が低い。何が大きな課題になっているか整理する。

---

## 1. 比較サマリー

| 観点 | 茂野 | 小山田 |
|------|------|--------|
| **受付突破率** | 高い | 低い |
| **接続後の獲得率** | 高い | 低い |
| **開き方** | 具体的な数字・用件を冒頭で伝える | 「人材紹介の件」と抽象的。確認が入りやすい |
| **接続後の主導権** | 相手の状況を理解しつつ、打ち合わせ・送付を提案 | ヒアリング中心。相手が主導になりがち |
| **細部の確認** | メールアドレス・担当者名を丁寧に | メールアドレス誤り（ゼニタカ）、担当者名聞きそびれ |

---

## 2. 受付突破率が低い要因（小山田の課題）

### 2-1. 開き方が抽象的で「取次する価値」が伝わりにくい

**茂野の開き方（例）**

```
・エアルエンジ：「電気工事士と施工管理の資格持ちの人材がいるんですけれども、中途採用のご担当様お手すきでしょうか」
  → すぐ「8名」も伝える

・プロサーチ：「二十名ほどいまして、東京、大阪、名古屋ですね、二十名ほどいまして 人材ご紹介できればなと思っていたんですけれども」
  → 冒頭で人数・エリアを具体的に

・東海EC：「人事担当の香坂様お手付けでしょうか」
  → 名前を出して依頼（フォローアップ時）
```

**小山田の開き方（例）**

```
・サンケイ・リーフテクニカ・アメリカン共通：
  「電気工事士や施工管理の人材紹介の件でご連絡したんですけれども。
   もしよろしければ、採用担当の方と少しお話できないかなと思いまして。」
```

**課題**

- 「人材紹介の件」だけだと、**何の用件か**が受付に伝わりにくい
- **人数・年齢・資格**などの具体的な情報がないため、「取次する価値」が弱い
- 結果、受付が「社名は？」「何の用件ですか？」と確認を入れ、その段階で断られるリスクが高まる

### 2-2. 受付突破の基本スクリプトとの乖離

`受付突破_断りパターンと繋ぎ方.md` の基本スクリプト：

```
「電気工事士で20〜30代の方、7〜8名ほどご紹介できる方がおりまして、
 いま人事ご担当の方いらっしゃいますでしょうか？」
```

**ポイント**

- 「7〜8名」という**具体的な数字**
- 「20〜30代」という**年齢の限定**
- 「ご紹介できる方がおりまして」＝ プッシュではなく**プル（情報提供の姿勢）**

小山田の開き方には、この**数字・年齢**が入っていない。

### 2-3. 確認が入りやすい＝取次のハードルが上がる

| 企業 | 小山田の架電で起きたこと |
|------|--------------------------|
| リーフテクニカ | 「社名様も一度お伺いしてよろしいですか？」 |
| アメリカン | 「エクスチェンジ様でよろしいですか？」「お名前もう一度伺ってもよろしいでしょうか？」 |

確認が入る＝受付が「誰か・何か」を判断する必要がある＝その時点で断られる可能性が高まる。

## 3. 接続後の獲得率が低い要因（小山田の課題）

### 3-1. 相手の「メリット」を先に伝える力が弱い

**茂野の接続後（例：東海EC）**

- 「三名おりまして、ぜひ面談までいけたらなと思っている」
- 「転職意欲が高い」「年収は合わせる」など候補者の強みを伝える
- 「30分程度、御社の魅力を知りたい」と打ち合わせを提案
- 相手の忙しさに配慮しつつ、次のアクションを明確にする

**小山田の接続後（例：リーフテクニカ）**

- 「採用ニーズとかそういうものを少しお聞きさせていただく時間をいただけないかな」
- → **こちらが聞く**姿勢が先に出る
- 相手が「紹介料はどれくらい？」と先に聞いてくる＝相手主導の流れ

**課題**

- 接続直後に「御社に合いそうな方が〇名いる」「〇〇のような強みがある」と**相手のメリット**を伝える前に、ヒアリングに入りがち
- 相手が「この人から紹介を受けたい」と思う前に、質問が多くなると、相手の関心が下がりやすい

### 3-2. ヒアリングの順序・バランス

小山田はヒアリングが**体系的で丁寧**（職種→勤務地→経験者定義→年収→出張→残業→休日など）。

一方で、

- **相手のメリット（人材の強み・人数・即紹介できる状態）**を先に伝える
- そのうえで「条件を聞かせていただければ、ピックアップして送ります」と**ヒアリングの理由**を明確にする

という流れにすると、相手の「話してよかった」という感覚が強まり、獲得率が上がりやすい。

### 3-3. 細部の確認ミス

| 事例 | 内容 |
|------|------|
| ゼニタカ組 | メールアドレス誤り（tky_が抜けてjinji@のみ）。メールが届かず、フォローアップで発覚 |
| ゼニタカ組 | 先日の架電で担当者名を聞きそびれた |

**課題**

- メールアドレスは送付前に**ホームページ等で正しい表記を確認**する（RA自身が裏で行う。復唱確認で十分）
- 担当者名は**その場で必ず聞き取る**（「お電話ごちそうさまは何様でしょうか？」など）

## 4. 大きな課題の整理（優先度順）

### 課題① 開き方：具体的な数字・用件を冒頭で伝える（受付突破率）

**現状**

- 「人材紹介の件でご連絡したんですけれども」と抽象的
- 人数・年齢・資格が入っていない

**改善**

- 冒頭で「電気工事士・施工管理の資格持ちで、20〜30代の方が〇名ほどおりまして、ご紹介できればと思い、お電話しました」
- 受付突破の基本スクリプトに合わせる

### 課題② 接続後：相手のメリットを先に伝える（接続後の獲得率）

**現状**

- ヒアリング（条件を聞く）が先になりがち
- 相手が「この人から紹介を受けたい」と思う前に質問が多くなる

**改善**

- 接続直後に「御社の条件に合いそうな方を数名、今すぐご紹介できる状態です」
- 「〇〇のような強みがある方がいます」など、人材の訴求を先に
- そのうえで「条件をお聞かせいただければ、ピックアップしてメールで送ります」とヒアリングの理由を伝える

### 課題③ 細部の確認：メールアドレス・担当者名（獲得の確実性）

**現状**

- メールアドレス誤り（ゼニタカ）
- 担当者名の聞きそびれ

**改善**

- メール送付前にホームページ等でアドレスを確認（RA自身が裏で行う。復唱確認で十分）
- 担当者名は必ずその場で聞き取る
- アドレスは復唱して確認する

## 5. 茂野から学ぶポイント（小山田向け）

| 茂野の強み | 小山田へのフィードバック |
|------------|--------------------------|
| 冒頭で具体的な数字（8名、20名）を出す | 開き方に人数・年齢を入れる |
| 「中途採用のご担当様お手すきでしょうか」と簡潔に依頼 | 「人材紹介の件」ではなく、用件を具体的に |
| 相手の忙しさに配慮しつつ、打ち合わせ・送付を提案 | ヒアリングの前に「御社に合いそうな方がいる」とメリットを伝える |
| 紹介料を先に提示し、相手の反応を見て調整 | 相手が聞く前に、料金の目安を伝える場合もある |
| メールアドレス・担当者名を丁寧に確認 | 送付前のアドレス確認、担当者名の聞き取りを徹底 |

## 6. 推奨スクリプト（小山田向け）

### 受付向け（開き方）

```
「お忙しいところ恐れ入ります。エクスチェンジ株式会社の小山田と申します。
 電気工事士や施工管理の資格持ちで、20〜30代の方が複数名おりまして、
 御社の条件に合いそうな方をご紹介できればと思い、お電話しました。
 採用担当の方、お手すきでしょうか？」
```

### 担当者接続後（最初の30秒）

```
「弊社、電気工事士や施工管理の20代・30代の方が複数名おりまして、
 御社の条件をお聞きして、合いそうな方を数名、メールでお送りできる状態です。
 もしよろしければ、勤務地や職種など、条件を少しお聞かせいただけますか？」
```
