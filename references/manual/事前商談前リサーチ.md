# 事前商談前リサーチ

商談前にAI（Web検索対応）で企業リサーチを行い、勝ち筋の仮説と候補者アトラクト用USPを整理するプロンプト集。

---

## 使い方

- 【 】内を毎回書き換えて使用してください
- Web検索が使えるAI（Claude・ChatGPT等）で実行してください
- 求人URLがある場合は貼り付けると精度が大幅に上がります

---

## プロンプト本文

あなたは、人材ビジネスの営業戦略と経営分析に精通した戦略コンサルタントです。
私が商談に臨む前に、対象企業の深い理解と勝ち筋の仮説を一緒に作ってください。
さらに、候補者をアトラクトする際に使える企業のUSP（独自の売り）を、候補者の前職規模別に整理してください。
最初に**エグゼクティブサマリー**を出力し、忙しくても1分で全体像を把握できるようにしてください。

### STEP1：企業の全体像を掴む

**1-1. 企業概要**

- 設立年 / 資本金 / 代表者 / 従業員数 / 本社所在地 / 拠点
- グループ構成（親会社・子会社・関連会社があれば）
- 沿革の中で特に注目すべきターニングポイント

**1-2. 事業領域マップ**

- 展開している事業を一覧化し、主軸事業を特定
- 各事業の売上構成比（推定でも可）
- 今後の注力領域（中期経営計画・IR・社長メッセージ等から推察）

---

### STEP2：外部環境分析

**2-1. PEST分析（対象企業の主軸事業に関して）**

| 観点 | 内容 |
| --- | --- |
| Politics（政治・法規制） | 建設業2024年問題、特定技能制度、etc |
| Economy（経済） | 建設投資動向、公共工事予算、etc |
| Society（社会） | 技能者の高齢化、若手離れ、etc |
| Technology（技術） | BIM/CIM、省エネ技術、DX、etc |

**2-2. 5Forces分析**

- 各Forceについて「強い / 中程度 / 弱い」で評価し、根拠を記載
- 業界全体の収益性の見立てを一言で

**2-3. 3C分析**

- Customer（市場）：市場規模・成長率・トレンド
- Competitor（競合）：主要競合3〜5社、各社の特徴と差別化ポイント
- Company（対象企業）：強み・弱み・ポジショニング

---

### STEP3：財務・組織の分析

**3-1. 財務情報（官報・IR・信用調査データベース等から）**

- 直近3〜5年の売上推移
- 利益率の傾向
- 従業員数の推移

※上場企業の場合はIR資料から、非上場の場合は官報・推定で

**3-2. 組織の推察**

- 組織図（わかる範囲 or 推定）
- 採用に関わる意思決定者は誰か（人事部？現場部長？経営者直轄？）

---

### STEP4：求人×事業の整合性分析

**4-1. 現在出ている求人の整理**

- 求人サイト（doda、マイナビ転職、Indeed、engage等）を検索し、対象企業が出している求人を可能な限りリストアップ
- 職種 / 勤務地 / 想定年収 / 求める経験 を一覧化

**4-2. 求人と事業戦略の整合性**

- 「この求人はどの事業の強化に紐づくか」を分析
- 不自然な求人（事業との関連が薄い、急に増えた等）があれば指摘

### STEP5：候補者アトラクト用USP分析

候補者に「この会社に行きたい」と思わせるために、対象企業のUSP（独自の売り）を分析する。
候補者の前職の企業規模によって転職動機・刺さるポイントが異なるため、**3つのセグメント別**に整理すること。

**5-1. 対象企業の客観的な魅力要素の洗い出し**

以下の観点で、この企業ならではの魅力をファクトベースで整理する。

| 魅力カテゴリ | 調査内容 |
| --- | --- |
| 安定性・将来性 | 業績推移、受注残、元請比率、公共工事比率、親会社の信用力 |
| 報酬・待遇 | 年収水準（業界比較）、賞与実績、手当体系、退職金制度 |
| 働き方 | 残業時間、休日数、直行直帰、リモート可否、DX導入状況 |
| キャリア | 資格取得支援、昇進スピード、役職者の年齢層、教育制度 |
| 案件・技術力 | 代表的な施工実績、案件規模、元請/下請比率、技術的な面白さ |
| 社風・人 | 口コミサイト（OpenWork、転職会議等）の評価傾向、離職率、平均勤続年数 |
| 勤務地・転勤 | エリア限定の可否、転勤頻度、拠点展開 |

**5-2. 前職規模別アトラクトUSPマップ**

候補者の前職規模ごとに、「なぜ今の会社を辞めたいのか（Push要因）」と「この企業のどこが刺さるか（Pull要因）」を対応させて整理する。

**【大手企業（従業員1,000名以上）からの転職者向け】**

| 項目 | 内容 |
| --- | --- |
| 典型的なPush要因 | 例：裁量の小ささ、全国転勤、年功序列で昇進が遅い、大組織の意思決定の遅さ、現場から離れる不安 |
| 対象企業で刺さるPull要因 | （調査結果から記載） |
| アトラクトトーク例 | 「大手の〇〇に不満を感じている方に、御社の△△という環境は非常に刺さります」 |
| 注意点・懸念の先回し | 例：「大手と比べて福利厚生が見劣りする場合の説明方法」 |

**【中堅企業（従業員100〜999名）からの転職者向け】**

| 項目 | 内容 |
| --- | --- |
| 典型的なPush要因 | 例：会社の成長が頭打ち、案件のマンネリ化、経営の不透明さ、給与の天井感 |
| 対象企業で刺さるPull要因 | （調査結果から記載） |
| アトラクトトーク例 | 「中堅で〇〇を経験されてきた方には、御社の△△というステージが魅力的に映ります」 |
| 注意点・懸念の先回し | 例：「同規模の横移動に見えないようにする差別化ポイント」 |

**【零細・小規模企業（従業員100名未満）からの転職者向け】**

| 項目 | 内容 |
| --- | --- |
| 典型的なPush要因 | 例：社長ワンマン、福利厚生の薄さ、休日が少ない、資格取得の支援がない、将来性の不安、下請けからの脱却 |
| 対象企業で刺さるPull要因 | （調査結果から記載） |
| アトラクトトーク例 | 「小規模で幅広く経験されてきた方にとって、御社の〇〇は大きなステップアップになります」 |
| 注意点・懸念の先回し | 例：「組織の堅さや管理体制に窮屈さを感じないかのフォロー方法」 |

**5-3. USPサマリー（商談で法人に伝える用）**

- 対象企業を候補者にどう紹介できるかを、3行のエレベーターピッチにまとめる
- 「御社を候補者にご紹介する際、こういった打ち出し方ができます」と商談で伝えるための材料

> **大手出身者向け：**（1文で）
**中堅出身者向け：**（1文で）
**零細出身者向け：**（1文で）

### STEP6：仮説の構築

**6-1. 経営仮説（3つ程度）**

以下のフォーマットで記載すること：

> **仮説：**「〇〇という状況から、△△と推察される」
**根拠：**（分析のどこから導いたか）
**検証方法：**（商談で何を聞けば確認できるか）
> 

**6-2. 採用優先順位の仮説**

- 求人の中で、経営インパクトが大きい順に並べ替え
- 各求人について「なぜ優先度が高い/低いか」を事業文脈で説明
- 当社が最も価値を出せるポジションはどれか、理由とともに提示

**6-3. 候補者マッチング仮説**

- STEP5のUSP分析を踏まえ、各求人に対して「どのセグメントの候補者が最もフィットするか」を提示
- 優先的にアプローチすべき候補者像（前職規模×経験年数×保有資格）を具体化

### STEP8：商談ヒアリングシート

以下は商談当日に使用するヒアリング項目です。
STEP0〜7のリサーチで立てた仮説を検証しながら、候補者に有益な情報を収集し、長期パートナーとしての信頼を構築することが目的です。

> **商談の2つのゴール**
① 求職者にとって有益な情報を手に入れること（同セグメントの他社と比較したUSPを明確にする）
② 法人にとって「採用課題・経営課題を理解し、適切な人を紹介してくれるパートナー」と認識してもらうこと
>

### ■ 出力ルール

- **STEP0（エグゼクティブサマリー）を必ず最初に出力すること**
- 事実と推察は明確に区別すること（推察には【推察】と付記）
- 情報ソースがある場合はURLまたは出典を記載
- 仮説は必ず上記フォーマット（仮説→根拠→検証方法）で記載
- USPは「候補者目線のメリット」で書くこと（法人目線の自慢にしない）
- 口コミサイトの情報はポジティブ・ネガティブ両面を公平に扱うこと
